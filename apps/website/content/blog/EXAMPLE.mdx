# Example post

This is a template post that shows how to create new blog posts. Keep this file as a reference and create new posts by following these instructions.

<Callout type="important">
  Blog posts now live in `apps/website/content/blog` and render through the
  Nextra-backed `/blog` route.
</Callout>

## Creating blog posts

<Steps>
  ### Create the file
  Copy this file (`EXAMPLE.mdx`) to create a new post.

### Rename it to the final slug

Use the filename as the public URL slug, for example
`my-new-post.mdx` becomes `/blog/my-new-post`.

### Update frontmatter and content

Fill in the metadata, write the body, and set `published: true` when the post
is ready to appear publicly.

</Steps>

You can still follow the normal workflow:

1. [Fork](https://docs.github.com/en/github/getting-started-with-github/fork-a-repo) and [clone](https://docs.github.com/en/github/creating-cloning-and-archiving-repositories/cloning-a-repository) the repository.
2. [Create a new branch](https://docs.github.com/en/desktop/contributing-and-collaborating-using-github-desktop/managing-branches) for your update.
3. Copy this file (`EXAMPLE.mdx`) to create a new post.
4. Rename the file to your desired URL slug.
5. Update the frontmatter and content.
6. Set `published: true` when ready.
7. Push your branch and open a pull request.

## Frontmatter reference

Every post must start with frontmatter in YAML format:

```yaml
---
title: "Your post title"
date: "YYYY-MM-DD"
author: "Author name"
published: true
tags:
  - release
description: "Optional summary used for metadata."
---
```
